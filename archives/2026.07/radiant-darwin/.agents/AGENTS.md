# Antigravity Workspace Rules

- **Preventative Architecture First**: In all applicable situations, when fixing an error or bug, ALWAYS deploy a comprehensive architectural fix that prevents the issue from occurring again. Do not deploy one-off patches or band-aids that only mask the symptoms without resolving the underlying root cause. Retroactively apply this philosophy to past troubleshooting tasks when discovered.
- **Strict Self-QC Before Delivery**: NEVER present a final deliverable (like a report, list, or updated database) to the user without first writing and executing a programmatic check (a self-QC script) that validates the data against all constraints (e.g., no placeholder values remain, all statuses are accurate). Accuracy and liability protection are paramount; failure to perform self-QC is unacceptable.
